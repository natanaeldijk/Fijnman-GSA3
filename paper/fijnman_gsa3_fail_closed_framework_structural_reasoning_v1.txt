\title{Fijnman-GSA³: A Fail-Closed Framework for Structural Reasoning}
\author{Natana\"el van Dijk}
\date{}

\begin{document}

\maketitle

\begin{abstract}
Modern AI systems often fail open: when input is underspecified or inconsistent, they resort to probabilistic completion, leading to hallucinations and unstable outputs. We introduce GSA³, a fail-closed reasoning architecture that replaces truth-based evaluation with structural admissibility under invariant constraints. A transition is performed only if it preserves all invariants and the claim is fully specified; otherwise the system refuses. Unlike classical SMT solving, GSA³ treats underspecification as a first-class failure mode, making refusal part of the semantics rather than an external solver condition. We formalize admissibility as a constraint over symbolic state spaces and extract maximal consistent substates via a kernel mechanism, yielding a hierarchy of admissible reasoning levels. We provide a deterministic implementation using SMT solving (Z3) and demonstrate that invalid states collapse to refusal rather than producing incorrect outputs. Our evaluation on controlled reasoning tasks shows that fail-closed semantics prevent hallucinations within the modeled symbolic domain by construction, under explicit specification constraints, offering a viable alternative to probabilistic completion in settings where correctness is critical.
\end{abstract}

\section{Introduction}
\label{sec:intro}

AI systems increasingly produce outputs under incomplete or ambiguous specifications, silently introducing hidden assumptions. Large language models (LLMs) hallucinate; symbolic reasoners often fall back on heuristics. The root cause is a shared design principle: \emph{optimize for completion, not correctness}. Given a dataset \(D\) and a method \(M\), it is common to observe \(M_1(D)\to O_1, M_2(D)\to O_2\) with \(O_1\neq O_2\). This instability is structural—it stems from permitting inference under under‑specification.

We take a different path: instead of improving inference under under‑specification, we prohibit it. We construct a \emph{fail‑closed} framework where reasoning is only allowed when all required components are explicitly specified. Missing definitions, ambiguous mappings, or competing interpretations are not approximated; they trigger a mandatory halt.

We argue that refusal is not a failure mode but a correctness condition in reasoning systems operating under incomplete information. Rather than replacing probabilistic models, GSA³ can be viewed as a complementary layer that constrains learned systems, ensuring that outputs are only produced when structurally admissible. We replace truth with \emph{admissibility} as the primary criterion. A statement is admissible iff it can be evaluated without introducing new assumptions. Reasoning becomes a sequence of admissible state transitions that preserve a set of invariants. Knowledge emerges as the intersection of all admissible continuations. This structure naturally yields a hierarchy of knowledge levels and eliminates hallucinations by design.

\begin{example}[Motivating example]
Consider an LLM asked: \textit{"What is the capital of a country that starts with 'Z'?"}
A typical system might guess \textit{"Zimbabwe"} or \textit{"Zambia"}.
GSA³ would instead return \textsc{Refusal}, because the input is under‑specified: there are multiple valid answers, and no unique solution can be structurally justified.
\end{example}

\subsection{Contributions}
\begin{itemize}
    \item \textbf{Fail-closed reasoning}: a framework where under‑specification leads to refusal, not guessing.
    \item \textbf{Layered symbolic core}: state spaces as logical formulas, invariants hierarchically ordered, kernel extraction via consistency checks.
    \item \textbf{Deterministic implementation}: reference implementation using Z3, with explicit handling of unknown as refusal.
    \item \textbf{Empirical validation}: experiments showing deterministic outputs, inconsistency detection, and graceful degradation.
\end{itemize}

\subsection{Why Not Probability?}
Probabilistic systems assign confidence scores to outputs, even when the underlying structure is incomplete or inconsistent. This leads to:
\begin{itemize}
    \item \textbf{Hallucinations}: high‑confidence answers that are structurally unsupported.
    \item \textbf{Silent failures}: no clear boundary between “low confidence” and “structurally invalid”.
    \item \textbf{Non‑determinism}: different runs may produce different outputs for the same input.
\end{itemize}
GSA³ instead enforces a hard boundary: if a conclusion cannot be structurally justified, the system refuses to produce it.

\section{Framework Overview}
\label{sec:overview}

GSA³ operates on a symbolic state space \(S = \{\sigma \mid \Psi(\sigma)\}\) defined by a formula \(\Psi\). Invariants are organised into a hierarchy \(\Phi_0\subseteq\Phi_1\subseteq\cdots\subseteq\Phi_k\) where \(\Phi_0\) are constitutional (must always hold) and \(\Phi_i\) (\(i>0\)) are epistemic refinements. A successor set \(S' = \{\sigma\mid\Psi(\sigma)\land\Xi(\sigma)\}\) is \emph{\(\Phi_i\)-consistent} if all its models agree on every \(\phi\in\Phi_i\). The kernel \(K^{(i)}(S')\) is \(S'\) when consistent, else \(\emptyset\).

The highest level with a non‑empty kernel determines admissibility:
\[
I^* = \max\{i\mid K^{(i)}(S')\neq\emptyset\}.
\]
If \(I^*\) exists, the system returns \(\textsc{Admissible\_Level}(I^*)\); otherwise \(\textsc{Refusal}\).

Modes per level are defined as:
\begin{itemize}
    \item \textsc{Strict}: exactly one model (uniqueness).
    \item \textsc{Safe}: multiple models, but \(\Phi_i\)-consistent.
    \item \textsc{Defeasible}: multiple models, not \(\Phi_i\)-consistent.
\end{itemize}
The mode profile \((m_0,\dots,m_k)\) is descriptive; admissibility is solely determined by \(I^*\).

Unlike classical SMT solving, GSA³ does not assume a well-posed problem. Instead, it explicitly models underspecification as a first-class failure mode and enforces refusal as part of the semantics, rather than treating solver failure as an external condition.

\section{Formal Core (Condensed)}
\label{sec:formal}

\subsection{Symbolic State Space}
\(S = \{\sigma\mid\Psi(\sigma)\}\) with \(\Psi\) in a decidable logic (e.g., QF\_LIA). \(\Psi\) must be fully specified; otherwise \(\textsc{Halt\_Spec\_Required}\).

\subsection{Invariant Hierarchy}
\(\Phi_0\subseteq\Phi_1\subseteq\cdots\subseteq\Phi_k\) with \(\Phi_0\) constitutional.

\subsection{Kernel}
\(K^{(i)}(S') = S'\) if \(S'\) is \(\Phi_i\)-consistent, else \(\emptyset\).  
The kernel is the maximal \(\Phi_i\)-consistent residue.

\subsection{Admissibility}
\(I^* = \max\{i\mid K^{(i)}(S')\neq\emptyset\}\). If \(I^*\) exists, output \(\textsc{Admissible\_Level}(I^*)\); else \(\textsc{Refusal}\).

\subsection{Fail-Closed Properties}
\begin{theorem}[Soundness]
If the system outputs \(\textsc{Admissible\_Level}(I^*)\), then \(S'\) is non‑empty and \(\Phi_i\)-consistent for all \(i\le I^*\).
\end{theorem}
\begin{theorem}[Determinism]
All decisions are deterministic given fixed \(\Psi,\Phi_i,\Xi\) and solver determinism.
\end{theorem}
\begin{theorem}[Monotonicity]
\(K^{(j)}(S')\subseteq K^{(i)}(S')\) for \(i<j\).
\end{theorem}

\section{Implementation}
\label{sec:impl}

We implement the core in Python using Z3. The class \texttt{GSA51Core} takes a list of variables, \(\Psi\), and \(\Phi_i\) hierarchy. Key methods:
\begin{itemize}
    \item \texttt{\_check\_consistency(xi, level)}: checks if \(S'\) is \(\Phi_i\)-consistent.
    \item \texttt{kernel(xi, level)}: returns \((nonempty, formula)\).
    \item \texttt{admissibility\_level(xi)}: returns \((I^*, kernel\_formula)\).
    \item \texttt{evaluate(xi)}: high‑level fail‑closed evaluation.
\end{itemize}
Unknown solver results raise \texttt{RefusalError}, which is caught and returned as \(\textsc{Refusal}\).

\section{Experiments}
\label{sec:experiments}

We evaluate GSA³ on four reasoning tasks that stress underspecification, inconsistency, and multi-model reasoning. While the tasks are intentionally minimal, they isolate core reasoning failure modes (underspecification, inconsistency, and ambiguity) that are known to underlie hallucination behavior in larger systems. For comparison, we simulate a probabilistic completion baseline representing typical LLM behavior, where underspecified or inconsistent inputs are resolved via heuristic or random completion. While simplified, this baseline captures the core failure mode: producing outputs under insufficient structural support.

\subsection{Setup}
All experiments run on a single thread with Z3 (QF\_LIA). We measure:
\begin{itemize}
    \item \textbf{Hallucination rate}: fraction of invalid outputs produced.
    \item \textbf{Determinism}: identical outputs across runs.
    \item \textbf{Refusal rate}: fraction of inputs leading to \(\textsc{Refusal}\).
\end{itemize}

\subsection{Discussion}
\textbf{Inconsistency}: Inputs like \(x=1\land x=2\) cause refusal. LLMs typically guess.  
\textbf{Underspecification}: Missing constraints lead to \(\textsc{Halt\_Spec\_Required}\); LLMs complete arbitrarily.  
\textbf{Multi‑model}: Multiple consistent models are reported as SAFE; LLMs may pick one arbitrarily, introducing bias.  
\textbf{Determinism}: GSA³ always produces the same outcome; LLMs vary.

These results show that fail‑closed semantics prevent hallucinations within the modeled symbolic domain by construction, under explicit specification constraints, and guarantee deterministic, verifiable behavior.

\section{Related Work}
\label{sec:related}

\paragraph{Probabilistic AI (LLMs)}: Optimize for likelihood, not correctness. They complete under‑specified queries with plausible but often incorrect answers. Unlike approaches that attempt to reduce hallucinations via better training or calibration, GSA³ eliminates them at the level of execution semantics.

\paragraph{Symbolic Reasoning (SAT/SMT)}: Provide correctness but typically assume a well‑posed problem. They do not handle underspecification gracefully; they either fail silently or rely on heuristics.

\paragraph{Epistemic Logic and DEL}: Model knowledge via possible worlds, but allow implicit completion. Our framework replaces possibility with admissibility and mandates explicit specification.

\paragraph{Fail‑safe Systems}: In control theory, fail‑safe means safe state after failure. Our fail‑closed means \emph{refuse} when specification is insufficient, preserving correctness.

\section{Applications and Limitations}
\label{sec:applications}

\subsection{Applications}
GSA³ is particularly suited for domains where incorrect outputs are more dangerous than refusal:
\begin{itemize}
    \item \textbf{Healthcare}: Diagnostic systems that must not hallucinate symptoms or treatments.
    \item \textbf{Finance}: Trading algorithms that should only act on fully specified market conditions.
    \item \textbf{Legal systems}: Contract analysis tools that must refuse ambiguous clauses.
    \item \textbf{Autonomous systems}: Decision-making pipelines where “I don’t know” is preferable to a wrong action.
\end{itemize}

\subsection{Limitations}
GSA³ is not a general-purpose reasoning engine. Key limitations include:
\begin{itemize}
    \item \textbf{No generation}: Proposals must be supplied externally; the system only validates.
    \item \textbf{No probability}: Uncertainty is modeled via kernel emptiness, not distributions.
    \item \textbf{Explicit specifications required}: The system halts if \(\Psi\), \(\Phi_i\), or \(\Xi\) are missing.
    \item \textbf{Performance}: Symbolic reasoning may not scale to very large state spaces without optimization.
\end{itemize}
These are not weaknesses but design choices: GSA³ is a \textit{filter}, not a generator.

\section{Conclusion}
\label{sec:conclusion}

We introduced GSA³, a fail‑closed reasoning framework that enforces admissibility under invariant constraints. By replacing truth with structural invariance and refusing under‑specified queries, we prevent hallucinations within the modeled symbolic domain by construction, under explicit specification constraints, and guarantee deterministic outputs. Our reference implementation demonstrates that fail‑closed reasoning is practical and offers a principled alternative to probabilistic completion in safety‑critical settings.

An important direction for future work is the integration of GSA³ with learned components, where neural models propose candidate states while the fail-closed layer enforces admissibility. This suggests a hybrid architecture combining statistical learning with structural guarantees.
